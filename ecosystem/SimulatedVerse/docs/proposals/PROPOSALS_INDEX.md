## Proposal Lifecycle

1. **Draft** - Initial SCP document creation
2. **Compile** - Transform RSEV → PUs + artifacts (`/api/proposals/compile`)
3. **Review** - Budget/entropy validation
4. **Publish** - Execute via PR workflow (`/api/proposals/publish`)
5. **Execute** - PU queue processing
6. **Land** - Merge and integrate

## Budget & Safety Rails

- **Token estimation** - Automatic cost calculation
- **Entropy gates** - Abort if system entropy > 0.7
- **Admin auth** - All mutations require bearer token
- **Quarantine not delete** - Safety-first file operations
- **PR workflow** - No direct filesystem mutations

## Quick Start

```bash
# Compile proposal (dry-run)
curl -X POST "$REPL_URL/api/proposals/compile" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d @proposal.json

# Publish proposal (execute)
curl -X POST "$REPL_URL/api/proposals/publish" \
  -H "Authorization: Bearer $ADMIN_TOKEN" \
  -H "Content-Type: application/json" \
  -d @proposal.json
```
