### **ANTI-THEATER ENFORCEMENT**

- Mock data generation without user request
- Documentation creation without explicit need  
- Endless refactoring without clear goals
- Tool building instead of using existing capabilities
- "Getting ready to get ready" vs actual progress

#### **Compelling Task Identification**
Priority order for autonomous grinding:
1. **Server crashes, build failures** (CRITICAL)
2. **TypeScript errors, test failures** (HIGH)  
3. **Linting warnings, TODO comments** (MEDIUM)
4. **Code style, optimization** (LOW)

#### **Evidence Requirements**
Every cycle must produce:
- Specific files changed with line counts
- Problems found and solutions applied
- Receipt with next recommended action
- System health verification
- Mobile compatibility confirmation
