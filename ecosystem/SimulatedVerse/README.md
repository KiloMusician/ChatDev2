## Git Boundary Notes

This repo has two known Git friction points on the current machine:

- it lives on a Windows-mounted worktree under WSL, so `git status` can be
  unusually slow even for small changes
- standard Git LFS hooks are installed, so missing `git-lfs` on PATH can block
  normal push / checkout / merge flows even when the content changes are fine

Operational rule:

- treat those as environment issues first, not as proof that the repo logic is
  corrupted
- use `npm run git:doctor` to distinguish hook/LFS problems from actual repo
  state
- use the wider workspace SCM truth tooling or Windows Git when WSL status
  checks stall on this mounted tree
