## Monorepo Structure
The project is organized as a monorepo, utilizing an event-bus-first communication strategy and autonomous orchestration. Key directories include `apps/` (core services), `clients/` (UI), `shared/` (common utilities), `content/` (directives), `kb/` (knowledge base), `analysis/` (data insights), `sidecar/` (LLM orchestration), `agent/` (self-improving automation), `scripts/` (development tools), `orchestrator/` (task orchestration), `.ops/` (autonomous operations), and `config/` (plans).

## Unified Context Management System
A central context management system (`src/context-management/unified-context-manager.ts`) integrates game state, consciousness, agents, tasks, user profiles, and the workspace environment. It ensures resource awareness, context-driven decision-making, and ethical considerations.

## Token Discipline Layer
This layer manages LLM usage with a local-first cascading approach, prioritizing Ollama (qwen2.5, llama3.1, phi3) and escalating to paid APIs (gpt-4o-mini) only when local confidence is low. It incorporates semantic caching, budget fences, and a symbolic planner.

## Event Bus Architecture
All system interactions occur via a WebSocket JSON-RPC event bus for real-time communication, supporting various event types and RPC methods. It includes Ajv schema validation and event logging.

## Data Storage Solutions
PostgreSQL, accessed via a Neon serverless driver, is the primary database, utilizing Drizzle ORM for type-safe operations and automatic schema migrations. Schema definitions are centralized in `shared/schema.ts` using JSONB for flexible game state storage. Session management also uses PostgreSQL.

## Game State Management
The system features a sophisticated game state with real-time updates and persistent saving to the database with configurable auto-save intervals. React Query synchronizes the UI with server state through optimistic updates.

## Authentication and Authorization
A simple player ID system currently links game state, with PostgreSQL-backed session management prepared for future user authentication. CORS and request validation are implemented at the API layer.

## UI/UX Decisions
The design emphasizes a mobile-first approach with a touch-optimized UI, large buttons, and a floating development console. Responsive panels ensure adaptive layouts across various devices, supporting one-hand operation.

## Core Gameplay Loop
The game incorporates an incremental/idle loop with a multi-tier progression, from manual resource gathering (Tier -1 Survival) to advanced automation (Tier 0 Basic Colony, Tier 1 Expansion). Resource management includes energy, materials, components, population, research points, tools, food, and medicine.

## Agent Framework (Modular Synth)
The system includes 14+ specialized agents accessible via GameShell navigation, ChatDev Console, and agent party orchestration. Agents produce measurable artifacts, state deltas, or bus events and are loaded via an ES module-based auto-discovery registry with contract validation, running alongside the autonomous PU queue processor.
