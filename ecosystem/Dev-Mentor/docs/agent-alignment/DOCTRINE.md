## V. THE ANTI-SPIDERWEB PROTOCOL

Spiderwebs form when:
- Many files are modified with no clear commit boundary
- Generated artifacts are mixed with source
- Context switching occurs mid-change

Prevention:

1. **Classify before acting:** Source / Config / Generated / Secrets / Docs
2. **Enforce hard rules:** Generated files ignored. Secrets never committed. Docs updated when behavior changes.
3. **Use containment:** Temporary branches. Stashes with meaning.
4. **End sessions clean:** Clean working tree, or one clearly-labeled stash, or a committed checkpoint.
