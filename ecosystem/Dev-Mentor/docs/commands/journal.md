# JOURNAL(1) — Terminal Depths Man Page

## NAME
  journal — query the NexusCorp system log (alias for journalctl)

## SYNOPSIS
  journal [OPTIONS]

## DESCRIPTION
  Access the system logs managed by the NexusCorp central logging service. This is vital for tracking agent movements and system anomalies.

## OPTIONS
  -f, --follow     output appended data as the journal grows
  -n, --lines <N>  show the most recent N lines
  -u, --unit <ID>  show logs for a specific system unit (e.g., nexus-relay)

## LOG SYSTEM
  NexusCorp uses a distributed logging system. While `journal` provides a unified view, some logs are only accessible on specific nodes or require root-level clearance.

## EXAMPLES
  journal -n 20
  journal -u nexus-relay -f
