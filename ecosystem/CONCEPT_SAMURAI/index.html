keeper.ps1

Lightweight, safe system orchestrator for Windows laptops. Switches between gaming and coding modes, reduces background interference, and monitors system health in real time — no bloat, no risky tweaks, just predictable performance.

PowerShell Windows Only Profile-Based Rollback Support -WhatIf Safe HTML Export

Mode Profiles

gaming

Optimizes for game performance by removing all dev interference and locking the power plan.

stops VS Code stops Docker shuts down WSL stops WSearch high-performance power Game Mode on

coding

Restores a balanced development environment.

balanced power restores WSearch Game Mode off WSL untouched

audio-safe

Kills all known audio-risk processes. Use when experiencing stutter or before running LatencyMon.

stops all dev tools stops browsers shuts down WSL stops Docker high-performance power Game Mode off

quiet

Maximum silence — stops all workloads and browsers. Use for recording, video calls, or deep troubleshooting.

stops all dev tools stops browsers shuts down WSL balanced power no Game Mode

diagnose

Balanced baseline for audio stutter testing without extra mode noise. Run doctor after switching.

balanced power no extra changes Game Mode off

restore

Rolls back the last reversible mode change using state/rollback.json.

reverts power plan restarts services reverts Game Mode

Commands

.\keeper.ps1 status Show current system health snapshot
.\keeper.ps1 mode gaming Switch to gaming mode
.\keeper.ps1 mode coding Switch to coding mode
.\keeper.ps1 mode audio-safe Kill all audio-risk processes (new)
.\keeper.ps1 mode quiet Stop all workloads, maximum quiet (new)
.\keeper.ps1 mode restore Roll back to previous state
.\keeper.ps1 watch Real-time health monitor (Ctrl+C to stop)
.\keeper.ps1 watch -DurationSec 60 Watch for 60 seconds then exit
.\keeper.ps1 doctor Diagnose Nahimic, audio risk, CPU, WSL
.\keeper.ps1 export Export JSON report to incidents/
.\keeper.ps1 export -Html Export HTML report to incidents/ (new)
.\keeper.ps1 prune Clean up old session and incident logs

Flags

FlagDescription
-WhatIfPreview changes without applying them (safe dry-run)
-DebugModeShow full structured action results for every step
-QuietSuppress INFO log lines
-HtmlGenerate HTML report when used with export
-DurationSec NStop watch after N seconds (0 = run forever)

Quick Start

# Preview gaming mode (no changes applied)
.\keeper.ps1 mode gaming -WhatIf

# Switch to gaming mode for real
.\keeper.ps1 mode gaming

# Audio is stuttering — kill everything that might interfere
.\keeper.ps1 mode audio-safe

# Run diagnostics to detect Nahimic, high CPU, WSL, Docker
.\keeper.ps1 doctor

# Export a self-contained HTML report
.\keeper.ps1 export -Html

# Roll everything back
.\keeper.ps1 mode restore

State & Storage

state/
  current.json      # Overwritten in place on every status/watch/mode change
  ringbuffer.json   # Rolling 180-sample buffer — oldest auto-dropped
  rollback.json     # Last reversible changes (power plan, stopped services)
sessions/           # Compact per-session summaries (max 30, pruned automatically)
incidents/          # Export bundles — JSON or HTML — auto-pruned after 14 days

Configuration Files

config/defaults.json

Global defaults: power plans, watch interval, ring buffer size, retention policy, doctor thresholds, launcher paths.

config/profiles.json

Mode definitions for gaming, coding, audio-safe, quiet, diagnose, restore. Edit to customize which processes and services each mode touches.

config/machine.local.json

Per-machine overrides (git-ignored). Copy from machine.local.example.json and add your process names, launcher paths, or profile tweaks.
