# ☁️ Cloud Systems Directory Context

## 📋 Directory Purpose
**Primary Function**: Multi-cloud orchestration, consciousness-aware scaling, cloud infrastructure management, and distributed computing for the KILO-FOOLISH ecosystem.

## ✅ Files That BELONG Here

- **Cloud Orchestrators**: Multi-cloud orchestration systems (`quantum_cloud_orchestrator.py`)
- **Scaling Systems**: Consciousness-aware auto-scaling and resource management
- **Cloud Adapters**: Cloud provider integration (AWS, Azure, GCP, etc.)
- **Container Orchestrators**: Kubernetes and container management systems
- **Service Mesh**: Cloud service mesh and microservice management
- **Infrastructure as Code**: Cloud infrastructure provisioning and management
- **Monitoring Systems**: Cloud resource monitoring and optimization

## ❌ Files That Do NOT Belong Here

- **Local Infrastructure**: Local system management belongs in `src/system/`
- **AI Systems**: AI orchestration belongs in `src/ai/`
- **Testing Files**: Belong in `tests/`
- **Configuration**: Cloud configs belong in `config/` or embedded
- **User Interfaces**: Cloud UIs belong in `src/interface/`

## 🔗 Integration Points

- **Consciousness**: Enhanced by `src/consciousness/` for aware scaling
- **Quantum Systems**: Uses `src/quantum/` for quantum cloud computing
- **AI Systems**: Provides cloud infrastructure for `src/ai/` systems
- **Security**: Integrates with `src/security/` for cloud security

## 🏷️ Required Tags

All files must include OmniTag/MegaTag headers with:
- **#CLOUD_SYSTEM** - Primary classification
- **#MULTI_CLOUD** - If supporting multiple cloud providers
- **#CONSCIOUSNESS_CLOUD** - If consciousness-aware
- **#ORCHESTRATION** - If orchestrating cloud resources
