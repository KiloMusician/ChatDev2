# Dependency Graphs for NuSyQ-Hub

## Extension Integration & Workflow

The following VS Code extensions and CLI tools are fully integrated for agent/human workflows:

- **Dependency Cruiser**: Generates JS/TS dependency graphs. Use the `depcruise` CLI or the VS Code extension to visualize and refactor module relationships. Output is `depcruise.svg`.
- **Graphviz Preview** & **Crabviz**: Preview `.dot` and `.svg` dependency graphs directly in VS Code. Open any graph file and select the preview tab.
- **Error Lens**: Highlights errors and warnings inline for rapid debugging. No configuration needed—enabled by default for all supported languages.
- **Todo Tree**: Aggregates all TODO/FIXME comments across the workspace. Use the sidebar view to track technical debt and agent/human action items.
- **Semgrep**: Provides static analysis and security scanning for Python and JS/TS. Run `semgrep --config=auto src/` or use the extension for inline results.

### Agent/Human Workflow

1. **Generate/Preview Dependency Graphs**: Use `depcruise` (JS/TS) or `pydeps` (Python) to generate graphs. Open with Graphviz Preview or Crabviz.
2. **Debug with Error Lens**: Review inline errors/warnings as you code. Agents can use this for automated error prioritization.
3. **Track TODOs**: Use Todo Tree to surface all actionable comments for both agents and humans.
4. **Run Static Analysis**: Use Semgrep for deep code quality/security checks. Integrate into CI or run locally.

All tools are pre-configured for this workspace. See `.vscode/extensions.json` for recommendations and `package.json`/`requirements.txt` for CLI dependencies.

This directory contains auto-generated dependency graphs for the NuSyQ-Hub
project. These are intended for both human and agent navigation, debugging, and
architectural enforcement.

## Files

- `depcruise.svg`: JS/TS dependency graph generated by dependency-cruiser +
  Graphviz
- `pydeps.svg`: Python dependency graph generated by pydeps + Graphviz

## Usage

- Open these SVGs in VS Code with the Graphviz Preview extension or any SVG
  viewer.
- Agents and developers can use these graphs to:
  - Visualize module relationships
  - Identify circular dependencies
  - Guide refactoring and debugging
  - Support automated navigation and code analysis

## Integration

- These graphs are referenced in docs/contracts/ and agent navigation workflows.
- Regenerate as needed after major refactors:
  - JS/TS:
    `depcruise --output-type dot src | dot -T svg > docs/graphs/depcruise.svg`
  - Python:
    `pydeps src/main.py -o docs/graphs/pydeps.svg --show-deps --max-bacon=2`
