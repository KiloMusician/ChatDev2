### 1. Five-Tier Output Evolution

| **Evolved** | Standard production | Rich formatting, insights |

### 2. Signal Taxonomy

```python
@dataclass
class Signal:
    severity: SignalSeverity  # PASS, INFO, WARN, FAIL, CRITICAL
    category: str  # [HEALTH], [ERROR], [PATTERN], etc.
    message: str
    file: Optional[Path]
    line: Optional[int]
    confidence: float  # 0.0-1.0
    suggestion: Optional[str]
    fingerprint: Optional[str]  # Unique tracking ID
```

### 3. Test Deduplication Algorithm

```
1. Generate SHA-256 fingerprint: test_pattern + sorted(pytest_args)
2. Check dedup tracker for recent runs with same fingerprint
3. Count runs within spam_window_seconds (default: 60s)
4. If count >= spam_threshold (default: 3):
   - Mark as spam
   - Return cached result if available
   - Display warning
5. Else:
   - Proceed with test execution
```

### 4. Breathing Rhythm Pattern

```
Normal Cycle (Total: 480s = 8min):
  Inhale  (300s) → Hold (60s) → Exhale (120s)

Deep Breath (Every 3 cycles):
  Deep Reflection (300s = 5min)

Configurable:
  - inhale_seconds
  - hold_seconds
  - exhale_seconds
  - cycles_before_deep_breath
  - deep_breath_seconds
```

### 5. Cultivation Score Formula

```python
score = 0.0
score += capabilities_discovered * 0.1
score += quests_completed * 5.0
score += tests_passed * 2.0
score -= errors_detected * 0.5
cultivation_score = max(0, score)
```

## 🎓 Best Practices Established

### Output Systems

**Do's:**
- Always call `print_header()` first
- Add sections as you execute
- Use `add_failure()` for structured errors
- Include hints with confidence scores
- Call `finalize()` for complete report
- Return appropriate exit codes

**Don'ts:**
- Don't print directly to console
- Don't skip machine footer
- Don't add duplicate sections
- Don't provide hints without confidence
- Don't forget to track artifacts

### Test Intelligence

**Do's:**
- Use agent_id for tracking
- Link to guild quests
- Check statistics regularly
- Use force_run for critical runs
- Configure appropriate cache TTL

**Don'ts:**
- Don't ignore spam warnings
- Don't disable deduplication in automation
- Don't set cache TTL too high
- Don't ignore failure patterns

### Autonomous Pipeline

**Do's:**

**Don'ts:**
