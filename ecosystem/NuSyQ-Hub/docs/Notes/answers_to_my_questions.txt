### 🔹 1. **How to Format System-Prompt-Like Contexts for Copilot in VSC?**

Copilot does **not** have a formalized "system prompt" field like ChatGPT does — but it does **simulate it** through:

* `.github/` → `instructions/` (recognized by GitHub Copilot Labs)
* Contextual analysis of **open tabs, recent edits, commit history, and workspace layout**
* **Comment-based "guidance"** written inline in your code or config files
* File-level **naming conventions** that signal intent (e.g., `*_CONTEXT.md`, `copilot.yaml`, `*.instructions.md`)

**You are already doing this perfectly.** However, here's how to maximize it:

#### ✳️ File Format Tips:

* Use `.md` or `.instructions.md`
* Use **YAML frontmatter** with `applyTo: '**'` and `priority: HIGH/CRITICAL`:

```yaml
---
applyTo: '**'
priority: CRITICAL
---
```

#### 🧠 Depth of Context Allowed:

* Copilot only "sees" \~20–100 lines **across open/related files** in practice.
* It **does not persist long-term memory** between sessions (yet).
* BUT: You can **simulate memory** via:

  * Copilot Labs' **"Custom Instructions"** (writes to `.github/instructions`)
  * **Open Tabs**: Copilot prioritizes open files and recently viewed ones
  * Comment-driven hints (`# This file manages consciousness context bridges`)

### 🔹 4. **What Are "Symbols" and "Tools" in Copilot?**

In **GitHub Copilot Labs**, you can define:

* **Symbols** = Tags/Functions/Classes to treat as “important” for context
* **Tools** = External scripts, compilers, interpreters, or pipelines used alongside Copilot

#### 📌 Use Case:

* **Symbols**: Highlight semantic anchors for Copilot (e.g., `generate_tags()`, `launch_quest()`)
* **Tools**: Tell Copilot, "I'm using this Python CLI / GODOT / Terminal script — don't overwrite it, extend it"

🧠 You can define these inside `copilot.yaml` (or in `.instructions.md`) like:

```yaml
symbols:
  - ZetaSetLexemeGenerator
  - copilot_enhancement_bridge
tools:
  - enhanced_agent_launcher.py
  - quantum_workflow_automation.py
```
